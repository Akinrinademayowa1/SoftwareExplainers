Getting started

Every mode in Skuul-CalcX shares the same underlying instrument: one display, one history log, and an input system designed around a simple idea — a wrong keystroke should be hard to make, not just easy to fix. This page walks through what's shared across every mode, then covers each one in full.

The display

  • Live preview. As you build an expression, a small preview line shows what it currently evaluates to, updating with every keystroke — you see your answer taking shape before you ever press equals.
  • Cursor navigation. In Standard, Scientific, and Programmer modes, tap anywhere inside a long expression to place your cursor there and edit that spot directly, instead of clearing and retyping the whole thing.
  • Copy. Hover or tap the result and a COPY button appears, sending the current answer straight to your clipboard.

History

Every calculation you complete, in any mode, is logged to the history panel — your most recent 30, newest first, each stamped with the time and which mode produced it. Tap any entry to bring its result straight back into the display. History is saved automatically and is still there the next time you open Skuul-CalcX on the same device.

Why some modes don't take direct typing

Calculus, Fractions, and Matrix mode all build their input through on-screen buttons rather than a free-typing keyboard. That's deliberate: a fraction typed as plain text is genuinely ambiguous (does 1/2+1/3 mean what you think it means, or did operator precedence just quietly change your answer?), and a misplaced parenthesis in a calculus expression fails in ways that are hard to spot. Building input from buttons that only ever produce valid syntax removes that entire category of mistake before it can happen. Each of those sections below explains exactly how their guided input works.

04

Calculus

Derivatives, integrals, limits, roots, and point evaluation — built with a guided keypad rather than free typing, so a stray character can't quietly turn a valid expression into an invalid one.

Building f(x)

Type your function using the dedicated keypad: digits, x, the four operators, parentheses, trig functions, logarithms, exp, abs, √, the constants π and e, plus x² for a quick square and xⁿ for any other power.

!

One naming note worth knowing: the ln button writes log( onto the screen. That's intentional — the underlying engine uses log() to mean natural log specifically, with log₁₀ kept as its own separate button for base-10. If you ever read the raw expression back, this is why.

Operations

  • d/dx — first derivative.
  • d²/dx² — second derivative.
  • ∫ₐᵇ f(x)dx — definite integral between two bounds. Bounds accept ordinary numbers, or the words inf / -inf for an integral that extends to infinity. The engine can correctly resolve integrals where the function itself shoots to infinity right at the edge of the interval — a case many calculators get wrong entirely.
  • lim x→c — the limit of f(x) as x approaches a point, including limits at infinity (enter inf or -inf as the target).
  • Roots — every real root of f(x) within a search range you specify.
  • Evaluate — plug in a specific x and get f(x) back directly.

Examples

f(x) = x², d/dx→2x
f(x) = exp(−x), ∫ from 0 to inf→1
f(x) = sin(x)/x, lim x→0→1
f(x) = log(x), ∫ from 0 to 1→−1
i

Answers are shown to 7 decimal places, not rounded off to look clean. If you're checking your work against a set of answer choices, you'll see the real value — something like 0.9998333 rather than a tidied-up 1 — so you can match it precisely.

05

Fractions

Chain together as many fractions as a problem needs, in any combination of the four operators, and get back one exact answer — never a rounded decimal standing in for it.

The fraction bar, not division

Fractions mode uses a dedicated symbol, drawn as a straight underline, purely to mean "this number over that number." It is never the same operation as ÷, which divides two already-built fraction values. Because the two are separate, an expression like 1_2+1_3 is never ambiguous — there's no reading of it that could mean anything other than one-half plus one-third.

The a_b button inserts a ready-made template with your cursor already positioned inside the numerator, ready to type — move to the denominator, and keep chaining with +, −, ×, ÷, and parentheses for as long as the problem needs.

Operations

  • Compute — build any expression and get back one exact answer.
  • Compare — put two expressions side by side and find out which is larger, smaller, or whether they're equal, with the cross-multiplication shown so you can see the work.
  • Decimal → Fraction — type any decimal and get its exact fraction back.

How answers are shown

Every result appears first as a simplified fraction. If it's improper, a second line shows the equivalent mixed number. A third line shows the decimal — with an = if it terminates exactly, or ≈ if it's a repeating decimal being shown to 7 places.

Examples

1_2 + 1_3 − 1_4→7/12
7_4 + 1_3→25/12
0.75 Decimal→Fraction→3/4

25/12 also displays as the mixed number 2 1/12, and as the decimal ≈ 2.0833333.

06

Matrix

Addition, subtraction, multiplication, scalar multiplication, transpose, determinant, and inverse — for any size from 1×1 up to 4×4.

Building a matrix

  • Set the size with the row and column steppers — the grid rebuilds instantly to match.
  • Tap any cell to select it, then type its value. Selecting a cell that already holds a value lets you overwrite it immediately, the same way clicking into a spreadsheet cell works.
  • Use the on-screen arrow buttons to move between cells without reaching back to tap each one individually.
