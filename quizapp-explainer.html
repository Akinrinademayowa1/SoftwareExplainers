What is it?

An exam system built for the real world

Schools need a reliable, zero-fuss way to conduct computer-based tests. SkuulQuiz PRO runs directly in any modern browser — no installation, no internet connection required after loading — making it ideal for any school computer lab.


Every piece of data — student records, questions, scores — is safely stored in an on-device database. Multiple admins each manage their own isolated datasets, partitioned by academic session and term, with no data bleeding between contexts.

How it works

Two portals. One seamless system.

1
Enter credentials
Students enter the Admin ID of their school's admin, their own matric number, select the academic session and term, choose their subject, pick the test type (First Test, Second Test, Third Test, or Exam), and enter their password to begin.
2
Pre-test dashboard
Before the clock starts, students see their school name, session, term, subject, test type, and any special instructions set by the admin for that specific quiz combination. Three golden rules are displayed as reminders.
3
Take the quiz
The quiz launches in full-screen mode. A live countdown timer turns red in the final 60 seconds. Answers are auto-saved every 5 seconds. The progress bar tracks answered questions. Questions can have optional images.
4
Results & retake flow
On submission, a 5-second redirect leads to the results screen showing a score ring, percentage, and an itemised correct/wrong answer review. Students blocked from retaking can send a retake request directly to their admin for approval.

1
Configure school settings
After logging in, the admin sets the school name, address, logo, active academic session and term. Subjects are then added and saved strictly under that session/term context — ensuring zero data overlap across periods.
2
Register students & assign subjects
Students are added with a name, matric number, and password. Each student is assigned to specific subjects from the current session/term subject list. All student records are scoped to prevent cross-term confusion.
3
Build the question bank
Questions are added per subject and test type (First Test, Second Test, Third Test, Exam). Each question supports four options, a correct answer, custom marks, and an optional image. Up to 200 questions per quiz combination. Per-quiz duration and instructions can also be set here.
4
View scores & export to ERP
The Scores tab groups results by subject with a sortable table. Each test type has a dedicated CSV export button — fully ERP-compatible (StudentID, StudentName, Subject, Score). Scores map to ERP Steps: First Test → CA1, Second Test → CA2, Exam → Step 3.

Features

Everything a school needs — out of the box

🗂
Session & term data isolation
Every dataset — subjects, students, questions, scores — is scoped to a specific academic session and term. Switching contexts automatically reloads only the relevant data, with zero cross-contamination between periods. A strict-fetch system prevents legacy data from bleeding into fresh contexts.
2024/2025 First Term Second Term Third Term
📤
ERP-compatible CSV export
One-click score export per subject and test type. Columns exactly match what the school ERP expects — StudentID, StudentName, Subject, Score. Built-in warnings flag ThirdTest incompatibility and scores exceeding ERP maximums.
🔁
Retake request system
Students blocked from retaking submit a request in-app. It appears in the admin's Logs tab with Accept / Reject buttons. Accepting clears only the specific score record — all other data is untouched.
👤
Multi-admin & role system
A Super Admin oversees all admins, controls the global student login gate, and can add/remove accounts. Standard admins manage their own isolated data. Each admin's data is completely independent of others'.
⏱
Per-quiz timer config
Set a custom duration and instructions for every Subject + Test Type combination — overriding the global default. Students see the exact instructions for their specific quiz on the pre-test dashboard.
🌙
Dark mode & school branding
Full dark / light theme toggle, persisted in the database. Admins can upload a school logo and name that appears in the topbar, quiz header, and student-facing screens — giving each school its own branded experience.
Architecture

Built from several code block modules

The application is structured as 19 logical modules, each responsible for a specific domain. There is no build step — the modules run directly in the browser as a single deployable unit.



§1
Storage Keys & State
Defines all database key names and the global Q2 state object holding the active admin/student session, quiz data, and context flags.
§2
Storage Utilities
Thin read/write/delete wrappers over the database. Handles JSON parsing, composite key construction, strict-fetch variants, and active context resolution.
§3
View Routing
Single-page view switcher — activates one of 7 views (Home, Student Login, Admin Login, Dashboard, Quiz, Results, Admin Panel) and manages topbar visibility.
§4
Dark Mode
Toggles the body.dark class, persists the preference in the database, and syncs both the topbar and home-page toggle buttons.
§5
Admin Authentication
Admin login validates against the admin registry by Admin ID + username + password. Session is persisted; logout clears it and returns to Home.
§6 & 6b
Student Authentication & Retake Guard
Student login validates against the session+term-scoped student store. Checks: admin exists, session/term/subject set, student found, password matches, not already taken. If blocked, opens the retake modal with an "Inform Admin" button.
§7
Student Dashboard
Populates the pre-test screen with school details, session/term, subject, test type, and per-quiz or global instructions before the student starts.
§8
Quiz Engine
Renders all questions from the question bank, manages the countdown timer, tracks answered/unanswered, auto-saves answers every 5 seconds, and requests full-screen mode.
§9
Results Renderer
Calculates score percentage, renders the animated score ring, labels performance (Perfect, Great Job, Good Effort, Keep Studying), and lists each question's outcome.
§10–11
Admin Panel & Settings
Loads the 7-tab admin panel. Settings module handles school name, logo, session/term, and the subject chip list — strictly scoped per context.
§12–13
Profile & Admin Management
Password changes, Super Admin controls (add/remove admins, global login gate), and role-filtered admin list view.
§14–15
Student & Question Management
CRUD for students (with subject assignment) and questions (with image upload support, marks, and answer validation). Both scoped to active session/term.
§16–17b
Scores, Logs & Retakes
Score records grouped by subject with CSV export; activity log (last 500 entries); retake request queue with Accept / Reject actions and badge indicator.
§18–19c
Modal, Helpers & SADM Controls
Reusable modal dialog, utility functions (escaping, time formatting, test type labels), Super Admin global gate, and the full admin data-reset procedure.
