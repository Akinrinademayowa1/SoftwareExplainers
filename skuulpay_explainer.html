What is SkuulPay

Two portals.
One complete system.

SkuulPay gives school administrators full financial control and gives students a clean, simple payment experience — all in one place.

Admin Portal

Manage students, sessions, terms, fee categories, class fees, enrollments, payments, and system configuration from a single dashboard with real-time analytics and audit trails.

Student Portal

Students log in with their ID, view outstanding fees, make payments through Paystack, and download their own PDF receipts and ledger — all from a clean, mobile-friendly interface.

Zero Infrastructure

Runs entirely in the browser. No installation required. Data is stored in a built-in database with full JSON export/import. Deploy by sharing a single HTML file.

Full Audit Trail

Every action — logins, payments, config changes, password updates — is recorded in both an Audit Log and an Activity Log, with timestamps and user attribution.

Workflow

From setup to
payment in minutes.

SkuulPay follows a logical, step-by-step flow. Set up once, collect all year.

1

Configure School Profile & Logo

Go to Settings → upload your school crest (auto-resized to 200×200px), enter the school name, address, and payment gateway keys. The logo immediately appears in the sidebar and will show on all PDF exports.

SettingsLogoPaystack Keys
2

Create Academic Session & Terms

Create the current academic session (e.g. "2024/2025") and its three terms. Mark the active session and active term — these drive all payment flows downstream.

Sessions & TermsAcademic Calendar
3

Define Categories & Class Fees

Create fee categories (Tuition, Exam Levy, Feeding, etc.) then assign exact amounts per class. JSS1 pays ₦45,000 tuition; SS3 pays ₦60,000 — configured once, applied automatically.

Payment CategoriesClass Fees
4

Register Students & Enroll

Add student records (name, class, student ID) then enroll them into the current session. Bulk enrollment lets you enroll an entire class at once.

StudentsEnrollmentsBulk
5

Monitor Payments & Ledger

The Payments module shows all incoming transactions with real-time Paystack verification status. The Financial Ledger shows every debit and credit across all students — filterable and downloadable as PDF.

PaymentsLedgerPDF Export
6

Communicate & Report

Send notifications to students or classes. Review the full audit trail. Export all data as JSON for backup, or export styled PDF reports for the principal or board.

NotificationsAudit LogPDF Reports

System Status

What's working,
what to know.

A candid summary of what's fully operational and what has dependencies.

Admin & Student LoginSession-based authentication with role routing. Admins use username + password; students use their student ID.
Full Data LifecycleStudents, sessions, terms, categories, fees, enrollments, payments, and ledger entries — all fully created, read, updated, and deleted.
PDF Receipt, Ledger & Payment ReportsAll three PDF exports generate correctly with logo, stat cards, colour-coded tables, and page footers.
Sidebar Navigation & SettingsAll 14 admin modules are accessible. Settings sidebar scroll fixed. School name wraps correctly on long names. Logo shows in sidebar and PDFs.
Financial Ledger Layout9-column ledger table uses correct min-width, responsive filter bar, truncated student/session cells with hover tooltips.
Paystack Live PaymentsRequires a live or test Paystack public key configured in Settings. Without it, the payment button will not proceed past the popup. Use a test key for demos.
Data PersistenceData is stored in the browser's built-in database. Clearing browser data or switching devices will reset the system. Use the JSON export regularly as a backup.
