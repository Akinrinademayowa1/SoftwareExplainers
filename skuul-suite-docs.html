The Suite

Each system is powerful.
Together they're complete.

Two independent tools, both built on the same philosophy — browser-native, zero-server, session-scoped data — engineered to hand off to each other at exactly the right moment.

📝
SkuulQuiz PRO
Computer-Based Testing

A fully self-contained, multi-admin online examination system built from several code block modules. Admins set up question banks and student rosters; students log in and sit timed, auto-graded quizzes.

  • Multi-admin with Super Admin oversight
  • 4 test types: First Test, Second Test, Third Test, Exam
  • Unlimited subjects and questions per admin
  • Timed auto-submit with countdown
  • Image-supported questions
  • Retake request queue with admin approval
  • Score export as ERP-compatible CSV
  • Full activity log (last 500 entries)
📊
Skuul-RMS
Result Management System

A complete result management system built for schools — from uploading CA scores to generating printable report cards in seconds. Processes raw CSV scores into beautifully formatted, position-ranked academic reports.

  • Session & Term Manager with one-click activation
  • CSV upload pipeline: CA1, CA2, Exam per subject
  • Auto grading, totals, positions, and remarks
  • Printable PDF report cards with school branding
  • Student self-service portal (no password needed)
  • Interactive analytics: bar, radar, doughnut charts
  • Cross-term performance trend tracking
  • Full timestamped activity audit log

SkuulQuiz PRO

The examination engine.

Everything needed to run a full computer-based school examination — question management, student management, live quiz delivery, and score export — packaged into several code block modules that run entirely in the browser.

📋
Question Bank
Build and manage questions per subject and test type. Supports image attachments, configurable mark values, and full answer validation. Scoped strictly per session and term.
🎓
Student Management
Add students with custom matric numbers, assign which subjects they sit, and manage credentials. All student records are partitioned by academic session and term — zero cross-period bleed.
⏱
Live Timed Quiz
Configurable countdown timer with auto-submit on expiry. Answers auto-save as students proceed. Questions served with optional image support. Up to 200 questions per quiz.
🔑
Multi-Admin with Super Admin Control
Unlimited admin accounts. Each admin manages their own isolated dataset — students, questions, scores — invisible to other admins. The Super Admin (SADM-001) can add or remove admins, view the full admin roster, and toggle a global student login gate that disables all student access system-wide with one switch. Role-filtered views ensure each level sees only what they need.
📤
Score Export (CSV)
One-click export per subject and test type. Output columns — StudentID, StudentName, Subject, Score — are ERP-compatible and feed directly into Skuul-RMS without modification.
🔄
Retake Request Queue
Students who are locked out after completing a quiz can submit a retake request. Admins see a badge indicator and can Accept or Reject requests individually. Full request log preserved.
📜
Activity Log
The last 500 activity entries are stored per admin — every login, student action, question change, and score event — giving full accountability without a server.

End-to-End Flow

From first question
to final report card.

The complete student assessment journey when both systems are deployed together — from setting up the exam to a student downloading their own PDF result.

01
SkuulQuiz PRO
Admin configures school, session, and subjects
The admin logs into SkuulQuiz PRO, sets the active academic session and term, adds subjects, and builds the question bank for each subject and test type.
02
SkuulQuiz PRO
Students are registered and assigned subjects
Students are added with matric numbers and passwords. Each student is assigned only the subjects they sit — they cannot see or access any other student's data or admin settings.
03
SkuulQuiz PRO
Students sit timed computer-based tests
Students log in using their Admin ID, matric number, session, term, subject, and password. They complete a timed quiz — answers auto-save, timer auto-submits on expiry. Results are stored instantly.
04
The Bridge
Admin exports scores as CSV
From the Scores tab, the admin exports results per subject and test type as a CSV file. The output format — StudentID, StudentName, Subject, Score — is exactly what Skuul-RMS expects for its CA1, CA2, and Exam uploads.
05
Skuul-RMS
Admin configures school identity in Skuul-RMS
School name, type, address, logo, and signatures are set. Sessions and terms matching those used in SkuulQuiz PRO are created and one combination is activated as the live context.
06
Skuul-RMS
CSV score files are uploaded per subject
The exported CSV files from SkuulQuiz PRO are uploaded into Skuul-RMS — CA1 for First Test scores, CA2 for Second Test, and Exam for the final exam — one file per subject. The system confirms the target session/term before writing.
07
Skuul-RMS
Reports are generated with one click
After all three assessment files are uploaded, Generate All Reports computes totals (CA1 + CA2 + Exam = 100), percentages, letter grades (A–F), teacher remarks, and class positions for every student simultaneously.
08
Skuul-RMS
Report cards are downloaded and distributed
The admin can filter by session, term, or class and download any student's PDF report card — carrying the school logo, signatures, subject breakdown, grades, and class position. Professional. Print-ready.
09
Skuul-RMS — Student Portal
Students access their own results independently
Without any password, students select their session and term, search by name or Student ID, and view their full results — bar charts, radar charts, grade breakdown, class position, cross-term trends, and improvement recommendations — all without involving the admin.
Data Architecture

Separate databases.
Shared philosophy.

Both systems store all data locally in the browser's database with no external servers. Each uses a structured key schema to ensure sessions, terms, and admins never collide.

📝 SkuulQuiz PRO — Database Keys
qapp_v2_admins_registry Global
cfg_settings_{adminId} Per Admin
cfg_subjects_{id}__{session}__{term} Scoped
cfg_students_{id}__{session}__{term} Scoped
cfg_questions_{id}__{session}__{term} Scoped
cfg_scores_{id}__{session}__{term} Scoped
cfg_logs_{adminId} Per Admin
retake_reqs_{adminId} Per Admin
✅ Scoped keys mean a student registered in 2024/2025 · First Term is completely invisible when the admin switches to Second Term. No accidental overwrites. No cross-session data leaks.
📊 Skuul-RMS — Database Keys
schoolConfig Config
sessionManager Config
schoolData[session][term][studentId] Scoped
adminCredentials Auth
activityLogs Audit
✅ schoolData is keyed hierarchically as [session][term][studentId] — First Term scores can never overwrite Second Term records, and historical data from any session is always accessible.
5 keys total. No sprawling key namespaces. The entire system — school identity, sessions, all student results, admin auth, and audit trail — lives in five clearly-named, purpose-specific entries.
