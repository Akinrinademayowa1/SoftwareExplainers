The Problem

Schools deserve better
than spreadsheet chaos.

Most schools still process results manually — copying scores across spreadsheets, calculating positions by hand, printing report cards one-by-one. It's slow, error-prone, and impossible to audit.

📊

Scattered Score Sheets

Teachers submit scores in different formats. Consolidating them into one coherent result takes days and introduces transcription errors.

🔀

Mixed-Up Terms

Without clear session/term boundaries, First Term scores accidentally overwrite Second Term records and historical data is lost permanently.

🔒

No Student Access

Students receive results on paper only — with no way to review their analytics, track improvement, or access previous terms.

🖨️

Report Card Drudgery

Filling individual report cards by hand — name, subject scores, position, grade, remarks — for an entire school is weeks of work prone to mistakes.

🔑

No Audit Trail

When errors appear, there is no record of who uploaded what and when. Accountability is impossible without a log of every action.

☁️

Cloud Dependency Risk

Cloud-based tools require internet access, monthly fees, and trust that a third-party won't expose sensitive student data.

How It Works

From CSV to report card
in four steps.

The entire result processing pipeline runs inside the browser. No login to a cloud service. No waiting for syncs.

01
⚙️

Configure the School

Set school name, type, address, logo, and signatures. Then create sessions (e.g., 2025/2026) and terms (First Term, Second Term…) in the Sessions & Terms Manager. Activate one combination to make it live.

→
02
📤

Upload Score Files

Enter the class name, then upload CSV files for CA1 (max 20), CA2 (max 20), and Exam (max 60) — one file per subject. The system confirms which session/term it is writing to before each upload.

→
03
🚀

Generate All Reports

After all three assessment types are uploaded, click Generate All Reports. The system calculates totals, percentages, grades, remarks, and class positions for every student simultaneously.

→
04
📄

Download & Distribute

From Manage Reports, filter by session, term, or class. Open any student's report card and download it as PDF. Students can search their own results in the Student Portal without admin involvement.

Two Portals

The right access
for the right person.

Skuul-RMS has two completely separate entry points — one for school administrators, one for students.

🔐 Admin Portal

Full Control
for Administrators

Protected by a username and password. The admin can configure everything, upload all results, manage records across all sessions, and generate report cards for any student.

  • Create and activate sessions & terms
  • Upload CA1, CA2, and Exam CSV files per subject
  • Generate all report cards with one click
  • Filter and manage all student records
  • Preview and download any report card as PDF
  • Delete or correct individual records
  • Change admin credentials securely
  • View full timestamped activity logs
🎓 Student Portal

Self-Service Results
for Students

No password required. Students select a session and term, search by their name or Student ID, and instantly access their own results — without admin involvement.

  • Filter results by session and term before searching
  • Search by name or Student ID
  • View subject scores, grades, and class position
  • Interactive bar, radar, and doughnut charts
  • Cross-term performance trend graph
  • Personalised improvement recommendations
  • View own report card (same format as admin)
  • Export full analytics report as PDF
Data Architecture

Private by design.
Structured by session.

All data lives in the browser's database — five distinct keys, each with a clear responsibility. No external server, no internet dependency, no third-party data exposure.

🏫
schoolConfig

School Identity

Stores school name, type, address, telephone, logo image, principal signature, and head teacher signature. Set once, used on every report card.

📅
sessionManager

Sessions & Terms

Tracks all academic sessions and their terms, which is currently active, and the full history of session/term combinations created by the admin.

📚
schoolData

All Student Results

Stores every student's scores, grades, position, and compiled results — keyed hierarchically as [session][term][studentId] so terms never overwrite each other.

🔑
adminCredentials

Admin Access

Stores the admin username and password. Default credentials are only written on first-ever load and preserved across refreshes — changing them in the Security tab sticks permanently.

📜
activityLogs

Audit Trail

A timestamped log of every action performed in the system — login attempts, uploads, configuration saves, report generation, and credential changes.
